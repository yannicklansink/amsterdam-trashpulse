# Contributing

Thanks for your interest in improving TrashPulse Amsterdam.

## Ground rules
- Keep issues focused and reproducible.
- Prefer small, reviewable pull requests.
- Discuss major changes in an issue before coding.

## Development
Scaffold and setup instructions will be added when the Next.js app lands.

## Submitting changes
1. Create a branch.
2. Open a pull request using the template.
3. Describe the change and any testing performed.

## Data usage
Use only public Amsterdam datasets and respect the terms described in `PLAN.md`.
